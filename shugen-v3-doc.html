QUADRANT ENGINE
象限诊断：四种情况

输入「r值达标率」和「当前门店库存」后，系统自动判断你处于哪个象限，并给出第一行动建议。

🔴 死循环
r值 < 80% AND 库存 < 2a
低库存正在压低r值，两个预警互为因果。先补货打破死循环，r值会自然回升，再查动销原因。
🟡 动销偏慢
r值 < 80% AND 库存 ≥ 2a
货架充足但卖得慢，是需求侧问题。查铺货率/陈列/竞品，考虑加大美顾或促销投入。
🟢 好问题
r值 ≥ 80% AND 库存 < 2a
卖得好但供应跟不上。加快补货频次，检查经销商出货节奏，防止断货损失。
✅ 状态健康
r值 ≥ 80% AND 库存 ≥ 2a
系统运转正常。聚焦升级条件：连续达标后申请升级，释放费用率空间。
⚠️ 为什么两个预警不能并列处理
库存低于2a（体温）和r值达标率不足（体质）是完全不同的信号。单看一个往往误判：库存低可能是因为卖太好（好事）或断货没补（坏事）；r值低可能是库存空了卖不出（死循环），也可能是真实动销不足（需求问题）。只有组合看，才能知道今天应该先打哪个电话。
CORE CONCEPTS
核心概念速查
💓 心跳公式
a = 心跳单元，一个周转周期T内的销售量
r = a ÷ T = 日销售率，是全系统通用语言

五层心跳：r₀消费者 → r₁门店 → r₂二批 → r₃经销商 → R全网
关系：R = Σr₁ = Σr₂ = Σr₃，必须等于消费者真实消耗
🌊 三层水位
零售门店： 满仓 3a → 预警 2a → 危险 1a
经销商： 满仓 2a → 预警 1a
二批商： 严格 1a 上限，超过即压货预警

货架 = 品牌领地。断货一次可能永久失去消费者。
🎓 年级系统
1年级 ×2.0激励 费用率50%：建认知·抢铺货·容忍亏损
2年级 ×1.5激励 费用率35%：扩渗透·提复购·回收投入
3年级 ×1.0激励 费用率20%：正常经营·靠口碑增长
4年级 ×0.7激励 费用率12%：收割利润·降费用

升级条件：r值连续3个月≥目标80%（2→3级需6个月）
原则：年级看r值，不看时间。
🔗 IPO0-13 全链节点
IPO0 原料采购 T60 断供防火墙
IPO1 工厂库存 T30 备货层
IPO2-4 经销商链 T30/14 推单节点
IPO5-7 二批商 T14 P2路径·严控1a
IPO8-10 门店链 T14/7 货架=领地
IPO11-13 消费者 T73 全链锚点
TIPS
使用技巧
🏷️ 上下文标签
输入框上方有四个标签：品牌 城市 定价 年份。点击填写后，AI每次回答都会自动带入这些背景信息，不需要反复说明。填写数据后自动同步品牌和城市。
📐 推演模式的用法
先填好基础数据（建立真实基数），再切到推演模式。AI会在你的真实R值基础上推演假设情景，而不是凭空假设。比如「如果渗透率从1.5%提到2%」，推演结果会基于你的实际门店数和经销商数计算。
🔄 多轮对话
樹根 v3 保留完整对话历史（当前会话内）。可以追问、要求重新计算、切换角度，AI记得你说过的所有内容。切换模式会插入分隔线，但历史不丢失。点「清空」重置对话。
