SKU 6 卡 schema spec V0.1

5 份契约共同的地基 · 每个 SKU = 合伙人节点 · 先设计、不写码 · 六月鲜 5-10 SKU 试点
编号 SKU-Schema版本 V0.1日期 2026-06-02署名 DS状态：schema 结构 · 待 DS 审
这是 JBP / 财务 / HR / 品类队长桥 / 经销商桥 五份契约都要写入的同一个底层 schema——没有它，五份全是无主孤魂。每个 SKU = 合伙人节点（关联多人收入，income_share ≤ 70%）。它的锚是树根（SKU 状态 / 晶体管），落在 IPO0-13 + A值 脊柱上。

节点身份（6 卡之前的根字段）

sku_id
主键
brand_bu
品牌/BU（试点 = 六月鲜）
name_spec
名称+规格（如 特级原汁酱油 500ml）
晶体管口径
状态 = SKU × 店/客户 × 时 × 场景（SKU 不是天然晶体管，落到这个粒度才是）；状态是分级/二元信号（有货/达标/动销）
partners[]
责任合伙人（多人）；income_share_pct ≤ 70%
双面 ref
欣和面 = 树根状态；零售商面/经销商面 = 品类角色/进销存角色（由两个桥写，可空）

6 卡字段结构（标谁写）

卡①角色共享·锚
role
enum{旗舰·主力·跑量·利基·防御·候选·退场}（名词·定身份）
role_reason
设定依据（战略地位/毛利/竞争）
set_by / date
谁设（品牌AI顾问/战略）+ 时间
⚠️ 命名硬规则：卡①「防御」= 名词·角色身份（如小康是防御型 SKU），不可与「防御中」（动作/某阶段在做的事）混用。
卡②生命周期阶段共享·锚
stage
enum{导入·成长·成熟·衰退}
stage_basis
判定依据（动销 r 趋势 / 渗透 / 增长率）
stage_since
进入该阶段的时间
⚠️ 阶段 = 评分的尺子本身：同样「动销 OFF」，导入期=正常、成熟期=病危。所有评分先看阶段。
卡③阶段目标JBP 写
target_r / 动销
目标日销率 / 动销（接 A值脊柱）
target_渗透/铺货/份额
该阶段该拨到的位置
period_T
周期（接水位算法）
jbp_ref
关联哪份 JBP / 品类计划 / 经销商计划
⚠️ 待你拍：卡③④⑤ 的度量是 SKU 级，还是 SKU×IPO 节点级？晶体管口径是「SKU×店×时×场景」，同一 SKU 在经销商面/零售面的动销·投入·产出都不同——节点维度不能压扁。
卡④阶段投入财务 写
budget / 费用流向
投多少 + 砸哪（陈列/试吃/种草/promo）
invest_ratio
投资比例（按阶段：导入大→成熟收）
红线 / 亏损上限
毛利红线（按生命周期加权）/ 战略·导入期亏损上限 (额度待 DS 定)
资金 / 账期
资金占用（钱脉络）
卡⑤阶段产出HR 写
actual_r / 渗透 / 动销
实际 vs 卡③目标
达标度
满分 = 该做的对的事做对（按卡②阶段尺子）
income
责任人收入结算（≤70% share）
cb / aiql
执行质量 / 客情评分
卡⑥退出条件共享·锚
exit_trigger
衰退且救不回 / 连续 N 周期未达 / 竞争失守
exit_action
转防御（降级保位）/ 退场
exit_owner
谁拍板（双签）

关系谁写哪张 · 投入有效性

共享锚
卡①角色 · 卡②阶段 · 卡⑥退出——三者共用，是评判的底座。
JBP
写卡③目标（下达「该拨到哪」）。
财务
写卡④投入（拨多少钱守红线）。
HR
写卡⑤产出（换回多少 → 算谁的钱）。
两个桥
写「双面 ref」的 counterparty 面（品类角色 / 经销商进销存角色）。
★ 核心
卡④投入 vs 卡⑤产出 = 投入有效性 = 因果入口（最深缺口）。

校验锚回脊柱（用脊柱图当尺子）

拿刚做的脊柱图 check 这套 schema 是否落在脊柱上——落上了：卡②阶段 = 生命周期门；卡③目标 / 卡⑤产出的核心度量 = A值 / r / 动销 / 水位（脊柱）；节点状态 = 树根 / SKU 状态（锚）。五份契约各写一张卡 → 都挂在同一个节点上 → 不再无主孤魂。没有飘。

试点六月鲜 5-10 SKU（宁愿慢）

先 1 个 BU（六月鲜）+ 5-10 个核心 SKU + 跑 6 个月，再全铺。选 SKU 建议覆盖不同角色 + 不同阶段（如 1-2 旗舰 / 2-3 主力 / 1 跑量 / 1 防御，分布在导入~成熟），这样才能验证 schema 对各种情形都成立，而不是只对一种。

边界诚实 · 待 DS 定

这是 schema 结构（字段定义），不是 code — 先纸面，审过再建表/写后端。
真实数据待接 — 卡③⑤的真实值靠信息脉络（汉询/POS，待接通）。
阶段判定 + 投入有效性的精确化要因果 — 现在阈值/归因都是占位，待因果模型。
政策待 DS 定 — income_share 占比、亏损上限额度、阶段判定阈值、退出触发的 N。
现状 — Layer 1.3，schema 是纸面地基。
