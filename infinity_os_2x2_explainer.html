象限 = 缺口分诊器

表面上所有"没达标"长得一样,但象限告诉你这是哪一种病——而病种决定了完全相反的动作。两条轴:做了没做(横)× 好不好(纵),这是你自己那张 2×2。

💡 为什么必须分象限(核心价值)
不分象限:所有缺口一刀切"降价/促销/调战略"→ 很可能在给一个根本没上架的SKU降价,纯浪费。
分了象限:每种病路由到对的动作——贵的因果模型只用在 B(7个)上,便宜的核查用在 C(64个)上,A 拿去学,D 回去修计划。这就是关差的"分诊台"。
🔍 我怎么判"做了没做"(诚实交代)
数据里没有"有没有真铺"的字段,我用代理:有实价 + 有销量 = 做了(它在某些店卖出去过=上过架);有目标却 0实价 0销量 = 没真做(铺货存疑)。
局限:这代理分不清"压根没上架"和"上了架但0动销"——所以 C 的第一步永远是"核查",到店看清是哪种,再定补铺/瘦身/退。
🎯 第二层:角色感知(叠在象限上)
象限先分出"这是执行差(C)";角色再分出"既然是执行差,该补 / 砍 / 退":
遵循自然 C → 补铺(成长担当·该在架) 经典 C → 砍(现金牛但SKU泛滥·瘦身) 轻 C → 退(在衰退·别盲目补)
所以"不是无脑全补"=象限说这64个是执行/铺货问题,角色再说这64个里哪些补、哪些砍、哪些退。两层叠加。
