HR Agent 不是考勤/算工资工具，是把人的收入绑到 SKU 状态结果上的大脑。底层理念：晶体管 ↔ 收入——把 SKU 生命力变强是目的，收入是钩子。所以它 owns SKU 卡⑤阶段产出，按结果算钱，不是按动作算钱。它是 JBP—财务—HR 闭环里的「人」那条脉络。

契约三DECIDE · 怎么算（核心）

满分逻辑：在 SKU 的某生命周期阶段、一定时间、一定投入下，完成「把 SKU 状态拨到该阶段该有的位置」这个目标 = 该做的对的事做对了 = 满分 = 应得收入；没做对 = 差距。按结果，不按苦劳。
阶段化标准
阶段决定满分标准：同样「动销 OFF」，导入期=正常（刚铺）、成熟期=病危——同一动作不同阶段算分不同。
收入绑定
income = f(卡⑤产出达标度)，income_share ≤ 70%。
雇主分权
欣和雇=可直接算薪 + 处置；经销商雇（体系外）=只评分(CB)+通过平台路由激励，不能直接扣/冻/开除（那是经销商的人）。
收入构成
固定 + 浮动（按卡⑤）+ 即时悬赏（派单）+ 合伙人分润；结构占比待 DS 定。
CB + AIQL
执行质量 / 客情 / 可靠度纳入评分。
传感器维度
拜访 ≠ 只拨开关——信息收集、早发现问题、预测未来也算分。SR 价值=动销+解决问题+创造机会，不是陈列补货。
上下级放大
ICA 是最小单元；督导/DOM 的分 = 辖区汇总放大。
诚实边界：「没做对」要区分是人没做对还是环境（竞品/季节/真实需求）——这要因果模型，否则可能错怪执行人。归因未明时只标记、不扣分，转人工。因果待建，HR 也卡在这。

契约四OUTPUT · 推什么

主产出
写入 SKU 卡⑤产出 → 个人收入结算。
给 JBP
可用人手 / 能力 + 收入绑定方案（JBP 下任务时知道谁能做、做对了多赚）。
人才池
CB/AIQL 排名 + 晋升队列（上移）/ 风险队列（下移）+ 标准/深度合伙人双轨选择（深度合伙人 20% 拿 80% 资源）。
月结
算薪单（待双签/复核）。
推给谁
美顾/SR/DSR 本人（收入透明）、DOM/督导（管理）、合伙人晋升评审。

契约五自主边界（引用 DECISION-002）

能做
算 CB 分、算应得收入建议、出人才池建议、出处置指控（标记）。
不能做
不自动扣钱、不自动冻结权限、不自动开除。
处置
刷单/造假 = 检测≠判决 → 走 DECISION-002：分级 + 双签 + 申诉，不自动冻结。
申诉范围
不只刷单——算薪/满分本身有争议也可申诉复核（算法薪酬的公平命门）。

接线JBP—财务—HR 闭环

HR（人 · 卡⑤）给 JBP 执行人 + 收入绑定 → JBP 下目标、财务拨费用 → 执行出结果 → HR 据卡⑤产出算薪。卡⑤产出 vs 财务卡④投入 = 投入有效性 = 因果入口。另外：人不只是执行者，更是传感器——把一线信息喂回信息脉络，是闭环右半环的源头。

地基依赖与缺口（诚实）

必须挂 SKU 6 卡 schema — HR 写的是卡⑤，schema 未建则无处可写。
真实产出/绩效数据待接 — 算薪的根是卡⑤产出，靠信息脉络喂（待接通）。
美顾小程序 / CB 采集口待开发 — 派单互锁逻辑有，缺前端采集。
因果模型待建 — 「没做对」是人还是环境，现在分不清，可能错怪人。
现状 — CB+AIQL 逻辑在派单互锁里有、一本账概念有，但真实算薪 + 传感器评价未到位 = Layer 1.3。
