L0 信号 → L1 入库 → L2 评估 → L7 推理+建议 → L8 资源申请 → L8 调度 → 真 SAP 单据号

                                                          Step 5 做的事

演示态使用 Mock SAP/钉钉,真生产时 TL 按本契约接真系统。

设计要点

① 接口契约先行

  • JSON Schema 定义清楚 9 状态 / 5 资源类型 / 路由规则
  • TL 看契约就知道怎么接·不用读代码
  • 真生产可以换实现·不动 Agent 上层

② Mock 是真能跑的 Mock

  • 不是空函数·真的模拟 SAP 5% 失败率
  • 真生成单据号(SAP_xxxx)
  • 行为接近真实 SAP 响应

③ 自主与对齐的工程实现

  • 阈值内 Agent 自主调度(快·不烦人)
  • 超阈值才打扰 DOM(对齐·把关大额)
  • 完美映射记忆 #19 审批流 V1.0 设计

④ 状态机严格

  • 终态(executed/rejected/expired)不可再变
  • 调用已终态的会被拒绝(防重复)
  • 失败可重试(failed → pending)

⑤ 完整可追溯

  • decided_at · executed_at · external_ref_id 都记录
  • 喂 L8 学习层用(下次 Agent 学"什么资源类型审批快")

        # 4. 金额阈值
        if auto_max is not None and amount <= auto_max:
            return {"mode": "auto", "reason": f"金额 {amount} ≤ 自动阈值 {auto_max}"}
