◆ 这份契约要锁死什么(做App前必须先定)

App不是先画界面,是先定业务契约:谁触发、收什么、算什么、推什么。根因"老是卡住"=先画界面后补逻辑;先定契约就不卡。

这套App = 既是数据采集口(INF-XD喂Agent)·又是价值推送口(INF-XM帮SR多赚)。一个App两个使命。

INF-XD 收 → Agent 算 → INF-XM 推(ITR举措+DSTE任务) → SR照做 → INF-XL 收入 → INF-XP 看状态 → 百人一步
★ 先锁角色(怕混层)

本App使用者 = GT零售业代 SR(欣和雇)。不是美顾(美顾=经销商雇导购·用美顾App)·不是DSR(经销商雇·用SFA)。同渠道不同角色,语义不同。本契约只管GT SR视角。

契约三 · DECIDE Agent算什么
3五层引擎(复用Step3/4·GT已跑通)L2门→L7行动
L2 判生命
读S01-S10过GT逻辑门 → PASS/FAIL/UNCERTAIN(没采集到不当失败)
L7①问题
哪个晶体管FAIL = 哪里出问题(如 S06动销FAIL = no_sellout high)
L7②原因
反推根因带confidence:老板不推荐0.85 / 陈列弱·不在主货架0.70 / 价签缺0.65 / 消费者认知0.50
L7③阶段
判生命周期(导入/成长/成熟)·同问题不同阶段·动作不同
L7④建议
输出该做的对的事(导入期:试吃/老板推荐/第二陈列/种草)
风险柱
数据先过"可信度门":存疑数据不计满分(防刷)·GPS/月末突击/幽灵库存四维·规则当黑脸
契约四 · OUTPUT 推什么(INF-XM输出)
4推给SR·该做的对的事这是目前最缺的!
INF-XM · ITR 方法/举措/措施
今天该做的具体动作 如:步步高西郊店·六月鲜导入期·老板未推荐(0.85)→ 今天去跟老板谈推荐+补第二陈列
INF-XM · DSTE 关键任务
战略解码下来的周期任务 如:本周重点SKU老板推荐覆盖目标5家·当前2家
INF-XL · 收入钩子
做对=多赚 如:救活"老板推荐OFF→ON"=+¥X·防断货=+¥Y(按SKU生命周期阶段·年级系数)
INF-XP · 个人状态
SR随时看自己 如:我管20家GT店·8家老板已推荐·5家待攻·本周收入进度
