JBP Agent · 业务契约 V0.1

联合生意计划「建议稿」生成 Agent —— 先纸面契约，审过再写界面/码
编号 JBP-Agent版本 V0.1日期 2026-06-02署名 DS状态：纸面草案 · 待 DS 审
JBP Agent = 为某渠道 × 某周期生成一份可进谈判桌的 JBP 建议稿（草案）。它建在 SKU 合伙人化 schema 上（卡③目标），靠 财务 Agent（钱）+ HR Agent（人）闭环，治理遵 DECISION-002——只出草案、不签约、不划扣。这是「唯一全新项」，本契约把它从演示原型推向真 Agent 的第一步。

契约一WHO · 谁触发

人工触发
大区 GM / KA 负责人 / SR 在年度·季度生意计划周期，对某渠道发起。
节律触发
系统在 JBP 周期节点（季度初 / 大促前 N 周）主动提示「该做某渠道 JBP 了」。
偏差触发
某渠道 r 实际 vs 对赌目标偏差超阈值 → 提示重做/修订 JBP。

契约二INPUT · 读什么（标来源）

历史盘子
该渠道上年 GMV / 动销 r / 渗透 汉询·POS
战略 SKU
清单 + 各 SKU 角色与生命周期阶段（卡①②③） SKU 6卡 schema
财务约束
毛利红线（22%）+ 可用费用池 / 年级费用率 财务 Agent
渠道模型
山姆=大包装/排他试吃 · 抖音=主播坑位+投流 · 便利店=小规格+收银台位 L3 渠道Agent
竞品价
竞品零售价 / 终端位置 价签识别 · A23 待建
审批阈值
客户 × SKU × 区域 三维阈值矩阵 DECISION-001

契约三DECIDE · 怎么算

选模型
按场景选 DECISION-001 五模型之一（战略 KA→E 分层 · 新经销商→C 双轨 · KA 大单→D · …）。
对赌目标
= 上年盘子 × 增量目标，受毛利红线约束。
费用流向
按渠道模型分配，优先砸向能拉动消费者「乐得买」的动作，砍无效进场费。
可解释
每个数字标依据（来自哪条 INPUT），不出黑箱数字。
红线：JBP Agent 只生成建议稿，不自动批准、不自动划扣。它产出的是谈判草案，不是生效契约。

契约四OUTPUT · 推什么

一份结构化 JBP 建议稿，四段：

【财务对赌】 目标 / 增量 / 毛利红线
【商品策略】 战略 SKU + 控价稳价
【费用流向】 按渠道模型的费用分配
【违约处理 → 转人工】 破价/未达触发预警 → 转人工裁定（不写自动划扣）

推给谁：建议稿推 KA 经理 / 大区 GM（去谈）；执行任务通过 INF-XM 下给 SR（SR 工作流随之从「拼方案」变「拿草案去谈+执行+喂回」）。

1
业务确认——KA 经理/GM 据现场情况修订草案。
2
双签——总部 + 大区 GM 双签（承 DECISION-001/002）。
3
绑定——录入 SKU 6 卡，绑定责任合伙人（income_share≤70%）。

契约五自主边界（引用 DECISION-002）

能做
生成 / 修订 JBP 建议稿；标依据；按偏差提示重做。
不能做
不签约、不拨款、不划扣、不自动冻结。终稿一律人工 + 双签。
违约处置
破价 / 对赌未达 → 走 DECISION-002 处置分级表：核实可逆（单人）→ 处罚不可逆（双签），转人工裁定，可申诉。

接线JBP—财务—HR 闭环 · 信息脉络

JBP（货脉络+促销 · 卡③目标）下达承诺 → 财务 Agent（钱 · 卡④投入）拨费用守红线 + HR Agent（人 · 卡⑤产出）派人绑收入 → 执行出结果（SKU 状态 / r 实际）→ 信息脉络回流校准下一轮 JBP。三者缺一，环合不上。

地基依赖与缺口（诚实）

必须挂 SKU 6 卡 schema — 否则又是无主孤魂（schema 本身待建，#28 宁愿慢的那块）。
需财务 + HR Agent 配套 — 没有钱脉络和人脉络配套，JBP 只是张纸。
数据/工具缺口 — 价签识别(A23)、KA 客户 CRM、账期管控、对赌追踪回收尚未建。
现状 — v1 控制台里有能出草案的原型（接真 API），但数据、双签接线、schema 均未到位 = Layer 1.3。
