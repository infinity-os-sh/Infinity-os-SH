品类队长桥 = 推力引擎。它是「双面 SKU」的接口层：把欣和的 SKU 经营（树根：状态/年级/流速）翻译成零售商的品类语言（品类角色/区隔/记分卡），用数据自动跑 ECR 8 步分析，帮零售商把品类做大 → 换回推力（位置/排面/promo/上架）。推力 ↑ → 位置 ↑ → 看得到 ↑ → 流速 ↑ → SKU 状态变强（回到树根）。

契约一WHO · 谁触发（关键：跑零售商的时钟）

节律触发
按零售商的品类评审周期跑，不是欣和的日历——品类评审 / 年度合同 / 上架·汰换窗口。
时间闸门
错过零售商的评审/上架窗口 = 0 机会，方案再好也白做。
联动触发
JBP 发起某零售商计划时调用——品类计划是 JBP 的内核。
偏差触发
某区隔 share / 利润落后记分卡 → 提示重做品类计划。

契约二INPUT · 读什么（标来源 / 待填）

全品类数据
含竞品的整个品类（区隔结构 / 各区隔 share·成长·利润）——这是和树根最大不同：树根只读欣和自己 零售商共享 / 第三方市场数据
欣和侧
欣和 SKU 的树根状态（年级/r 流速/状态）= 双面接口的欣和面 树根
购物者视角
购物者在该品类的决策树（怎么找/怎么选）+ 消费者购买频率·价值偏好——角色要从零售商和消费者双视角定义 待填/调研
零售商意图
买手/品类经理的品类目标·评审重点·角色判断——过去失败的盲点。靠 KA 经理当「零售商意图传感器」+ 联合生意计划 + 数据共享采集 待填/关系
品类角色定义
该零售商怎么定品类角色（特色/优先/基本/季节/便利） 零售商专属·待填
记分卡 KPI
该零售商品类记分卡考哪些（share / 现金毛利 / 客流 / 周转） 待填
货架规则
planogram / 最小排面 / 最高 DOS 规则 待填
欣和现状
在该零售商是否品类队长、现有 share / 位置 待填

契约三DECIDE · ECR 8 步（落成机制）

1
品类定义——和零售商事先共识品类范围。
2
品类角色——特色/优先/基本/季节/便利，从零售商 + 消费者双视角定义（与定义一样，事先与零售商共识），映射欣和 SKU 年级。
3
品类评估——品类 → 区隔 → SKU 三层，区隔级气泡（share vs 成长，泡=规模），对标竞品 + 市场均值，找落后区隔 + 利润机会。
4
记分卡——零售商的 top-line 目标（share + 现金毛利），铁律「要份额，绝不丢毛利」。
5
品类策略——按区隔分配：引流 / 增交易 / 产现金 / 产利润。
6
四杆战术——商品组合（SKU 合理化：补强、删弱，含自家尾货）· 价格（主力品项 competitive 无毛利损失）· 促销（频率对齐市占，估计→衡量）· 货架（DOS + 最小排面 → 删减，余位给策略 SKU）。
7
调整计划——累加各战术对销售/利润的影响 vs 记分卡目标 → 调整 → 与零售商共识。
8
检视——实际 vs 计划、分析偏差、重启循环（品类管理是过程，不是任务）。
全程从零售商视角算，不是推欣和的货。产出是「给零售商的品类计划建议稿」。战术→销售/利润的影响现在只能「估计→衡量」（讲义原话），精确因果待因果模型。
分支：欣和是不是品类队长？这决定打法。是队长（某品类欣和领先）→ 替零售商跑完整 8 步。不是队长（如酱油可能海天是队长）→ 打法变成：影响队长的品类方案、做有数据的强势贡献者、用一两个过硬区隔逐步争取队长资格。契约二「欣和现状·是否队长」决定走哪条路。

契约四OUTPUT · 推什么

主产出
给零售商的品类计划建议稿（角色 / 记分卡 / 区隔策略 / 四杆战术）——谈判用。
回流欣和
哪些自家 SKU 该补强、卡哪个区隔位 → 喂树根 SKU 状态，指导财务（卡④往哪浇）/ HR（卡⑤奖谁）。
给 JBP
品类计划 = JBP 的内核（JBP 对赌从「欣和量」重构成「品类增长 + 欣和份额」）。
推给谁
KA 负责人 / 大区 GM 去和零售商共识；终稿 = 零售商共识 + 欣和侧双签。

契约五自主边界（引用 DECISION-002）

能做
跑 8 步分析、出品类计划建议稿、出 SKU 合理化建议（含建议删自家弱单品）。
不能做
不自动改零售商陈列/价格/汰换、不自动建议下架、不替零售商决策——汰换走和零售商的共识 + 检视。
队长信任红线（守护者非控制者，在零售商关系的应用）：只推自家货、乱删别家的品类队长会丢信任、丢队长资格（讲义第 11/44 页）。当公正的品类管家，才长期有推力。这和 DECISION-002 是同一个共创逻辑。

接线双面 SKU · 锚回树根

双面 SKU：欣和面（树根：状态/年级/流速）↔ 零售商面（品类角色/区隔/记分卡贡献）。推力生在这个接口。品类桥不是第二棵树，是让零售商也愿意往同一棵树浇水：推力 ↑ → 位置 ↑ → 看得到 ↑ → 流速 ↑ → SKU 状态 ↑（回到根）。产出回流喂树根、指导财务/HR、喂 JBP。节奏跑在零售商的评审时钟上。财务握手：为争品类而「投 margin / 降价」的建议，须经财务 Agent 的红线 + 亏损上限校验（品类桥提议，财务守闸）。

地基依赖与缺口（诚实）

全品类数据（含竞品）= 最大数据缺口 — 树根只有欣和自己的；品类管理要全品类，靠零售商数据共享或第三方市场数据（待接）。
零售商专属规则节奏 = 待你给资料填 slot — 山姆/盒马/大润发… 的角色定义 / 记分卡 KPI / 评审节奏 / planogram 规则。
因果模型 — 「战术→销售/利润」现在只能估计→衡量，精确归因待建。
挂树根 + DECISION-002 — 双面接口的欣和面接树根，治理接 DECISION-002。
现状 — 框架级 V0.1，零售商血肉待填 = Layer 1.3。
