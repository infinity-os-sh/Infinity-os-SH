SECTION 02
启动前需要准备的三样东西
这三样东西准备好，发给开发团队，他们就可以开始接入。
1
企微管理后台的三个参数
你来拿

登录 work.weixin.qq.com → 管理后台

进入：应用管理 → 自建应用 → 创建应用
应用名称：库存上报
上传图标，选择可见范围（上海业务员）

创建后拿到三个参数：

CREDENTIALS · 需要的三个参数
企业ID（corpid） ← 我的企业 → 企业信息
应用ID（agentid） ← 创建应用后自动生成
应用Secret ← 创建应用后自动生成，只显示一次，请保存

把这三个参数交给开发团队，他们用来配置企微登录和数据提交。

2
企微多维表格
你来建

在企业微信里新建一个多维表格，按以下列名设置：

列名类型说明
上报时间日期时间自动填入
业务员姓名文本企微登录自动带入
门店名称文本SFA门店库自动带入
门店等级单选S+/S/A/B/C/D
SKU单选自动带入
货架库存（瓶）数字业务员填写
仓库库存（瓶）数字自动换算
合计库存（瓶）数字自动计算
实际销速r数字系统自动算
目标销速r数字树根项目数据
达成率数字自动计算%
可卖天数数字自动计算
销速状态单选超标/正常/偏低/严重不足
异常标记多选压货/促销/窜货
照片附件货架照片
备注文本业务员填写

建好后，把表格链接发给开发团队，他们配置API写入权限。

3
预警机器人群
你来建

在企业微信新建一个群：上海库存销速预警

成员：上海区域经理 + 相关负责人

在群里添加一个群机器人，复制Webhook地址发给开发团队。

开发团队配置好后，以下情况自动推送消息到这个群：

TRIGGER CONDITIONS · 触发条件
达成率 < 60% → 🔴 严重预警推送
达成率 60-90% → ⚠️ 偏低预警推送
达成率 > 110% → 🚀 超额完成推送
可卖天数 < 3天 → ⚡ 紧急补货提醒

KEY API ENDPOINTS · 关键接口
# 1. 获取企微用户身份
GET https://api.weixin.qq.com/sns/jscode2session
?corpid=YOUR_CORPID&agentid=YOUR_AGENTID&js_code=CODE

# 2. 写入企微表格
POST https://qyapi.weixin.qq.com/cgi-bin/wedoc/spreadsheet/append_rows
Authorization: Bearer ACCESS_TOKEN
Body: { sheet_id, rows: [{ 上报时间, 业务员, 门店, 实际r, 达成率... }] }

# 3. 推送预警消息
POST WEBHOOK_URL
Body: { msgtype: "markdown", markdown: { content: "预警内容" } }
