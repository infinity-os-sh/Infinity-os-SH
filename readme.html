品类管理包 · README

用途:把零售商的 ECR 八步品类管理,变成欣和自己的"渠道品类管理成长 Agent"。本包=六月鲜跑通的全部成品 + 可复用脚本 + 方法说明。
另开对话做别的品牌时:把本说明的 GitHub 链接发给 Claude,即可立刻接上打法。

一、方法核心

一循环

ECR 八步:①定义 ②角色 ③评估 ④评分表 ⑤策略 ⑥战术(范围/价格/促销/货架) ⑦执行 ⑧检视。

两账

账X=欣和自己的SKU记分卡(数据我们有)/ 账A=零售商品类记分卡(需竞品·品类大盘,我们缺)。

2×2 分诊器(关差先分诊)

X=做了没做(执行;有实价+销量>0=做了)· Y=好不好(达成率)

象限含义动作
A 做了×好投入有效·正样本抽成功模式复制
B 做了×差投入无效唯一需因果模型·贵·养
C 没做×差执行问题核查+补铺·便宜·最常见
D 没做×好设计/预算错反向校验计划

摆位:左上D / 右上A / 左下C / 右下B。

三级视图

品牌组合象限(看子品牌站位)→ 子品牌SKU象限(看SKU分布)→ 品类×规格 热力网格(看缺哪个规格,推荐主用)。

硬规则:零售商5类品类角色(特色/优先/基本/季节/便利)≠ 欣和7类SKU六卡角色(旗舰/主力/跑量/利基/防御/候选/退场),不可混。

二、配色规范(白底)

品类色:生抽 鲜味 红烧 老抽 蒸鱼 有机

达成色:≥100 50-99 <50 没动
