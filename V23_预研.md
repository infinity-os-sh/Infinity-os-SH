## 🎯 V23 的核心命题

**V22 把战略推到了"全局三场景"。V23 要把它推到"局部精细化"**。

具体表现为四个升级方向：

1. **城市群虹吸效应**：上海做大，杭州/苏州被抢流量？
2. **大区独立场景**：华东保守、西北激进，每大区不同风险偏好
3. **合纵 7 棒接力**：渠道间的真实接力顺序与流量传导
4. **促销动态注入**：促销作为时间事件嵌入 V22 节奏曲线

## 一、城市群虹吸效应建模

### 当前 V22 的盲点

V22 假设 150 城**互相独立**——每个城市的目标独立计算，加起来 = 总目标。

**这在物理上不成立**：
- 上海 2029 年增长 30%，**周边的苏州/无锡/嘉兴会被分走客流**
- 北京放大 → 廊坊/天津部分需求被吸走
- 成都激进 → 重庆部分高端需求外溢

### 数学模型雏形

```
city.actual_target(y) = city.raw_target(y) × (1 - siphon_loss)

siphon_loss = Σ (邻近高吸引力城市的吸引强度 × 距离衰减)
            = Σ (anchor_city.target_growth × influence_radius / distance)
```

### 数据需求

- 城市间地理距离矩阵
- 城市间高铁/公路通达性
- 历史虹吸数据（如有）
- 锚点城市识别（一线/新一线）

### 实现优先级
**中**——可用，但**前提是有真实虹吸数据校准**。否则只是漂亮的数学游戏。

## 二、大区独立场景比例（V21 中你提过的"B 方案"）

### 当前 V22 的局限

V22 三场景是**全局**的——保守/主案/激进只能整盘切换。

但战略上完全应该：
- **华东**用"保守 80%"——基础牢，稳健推进
- **西北**用"激进 130%"——市场窗口短，必须抢
- **华南**用"主案 100%"——平衡

### V23 设计

每个大区独立设场景比例：

```
region_factor = {
  '华东': 0.85,
  '华北': 0.95,
  '华南': 1.00,
  '华中': 1.10,
  '西南': 1.15,
  '西北': 1.20,
  '东北': 0.75
}

city.target(y) = base.target(y) × region_factor[city.region]
```

### UI 设计

- 顶部场景切换 → 改为"大区设置"按钮
- 弹出 7 大区独立滑块
- 整盘 Σ 实时校验

### 实现优先级
**高**——这是 V21 反馈最可能要求的功能。

## 三、合纵 7 棒渠道接力

### 概念回顾

7 棒 = 不同 Place 路径下的 People×Promotion 组合：
1. 小红书种草
2. 抖音爆发
3. 电商转化
4. O2O 即时
5. 仓储会员
6. 大卖场
7. 社区超市

### V23 接入思路

每个城市目标按 **7 棒占比**拆分：

```
city.channel_split[y] = {
  '小红书': 0.05,
  '抖音': 0.12,
  '电商': 0.20,
  'O2O': 0.10,
  '仓储': 0.08,
  '大卖场': 0.30,
  '社区': 0.15
}

city.target_per_channel(y, ch) = city.target(y) × channel_split[ch]
```

### 接力顺序的可视化

7 棒不是平行的，是**按时序接力**的：
- 1 月：小红书种草启动
- 2-3 月：抖音爆发跟进
- 3-4 月：电商承接转化
- 5-6 月：O2O/仓储补量
- 全年：大卖场/社区铺货

V23 应该有一个 **7 棒×12 月 的甘特图**，展示某城市的渠道接力节奏。

### 数据需求

- 各渠道历史销量占比
- 渠道间转化率（小红书曝光 → 抖音观看 → 电商成交）
- 各渠道单位投入产出

### 实现优先级
**中**——视觉效果好，但**对实际运营帮助取决于数据是否真实**。

## 四、促销动态注入

### 当前 V22 的盲点

V22 节奏曲线是**平滑**的——年度按 25% / 40% / 60% / 82% / 100%。
**真实业务不平滑**——618、双 11、春节、中秋会形成尖峰。

### V23 设计

引入"促销事件"概念：

```
promo_events = [
  { date: '2026-06-18', name: '618', uplift: 0.15, channels: ['电商', 'O2O'] },
  { date: '2026-11-11', name: '双11', uplift: 0.25, channels: ['电商'] },
  { date: '2026-02-09', name: '春节', uplift: 0.30, channels: ['大卖场', '仓储'] }
]

monthly_volume(month) = base_monthly × (1 + Σ promo.uplift)
```

### 蝴蝶效应联动

V22 已经有"蝴蝶效应 B 类失真 = 促销扰动"的概念。
V23 把促销显式建模后，蝴蝶效应 Agent 就能：
- 区分"真增长"和"促销带动的虚增长"
- 促销结束后的回落预警
- 跨年同期比较时剔除促销因素

### 实现优先级
**高**——这是真正连接战略目标和业务运营的关键钩子。

## 五、其他待评估方向

### A. 竞品动态嵌入
- 海天/李锦记的市占变化作为外生变量
- 当竞品突然降价时自动触发场景重算

### B. 现金流时间错位建模
- 你 memory 里强调的"钱流与货流时间错位"
- V22 当前只算年度预算，没算月度现金流

### C. 经销商承接能力约束
- 上游约束已有"工厂产能"
- 但**经销商也是约束**——不是无限承接
- 需要按经销商配置容量上限

### D. 美顾/DSR 团队人头数模型
- V22 团队约束是粗放的"+30%/年"
- V23 应该按城市精细化：上海需要多少美顾、济南需要多少
