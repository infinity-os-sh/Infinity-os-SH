渠道品类管理成长 Agent

八步骨架 Spec

把 ECR 八步法(零售商的盈利-增长模型)结构化 → 建模 → 工程化,做成欣和的渠道品类 Agent。原则:一个循环 · 两本记分卡 · 一组对应门,既满足零售商,也满足欣和,双方各取其值。
署名 DS · 状态:纸面设计待审 · 不是 demo / 不是代码
一个循环 · 两本账 · 一组对应门 ≠ 两个 Agent 再结合

不建第二套循环,不"分开建再结合"(那是整合补丁陷阱)。一个八步循环跑两本记分卡:零售商品类记分卡 + 欣和 SKU 记分卡。中间用「对应门」在每一步把两本账翻译对齐。"全方位 Agent" 不是最后单独建的第三个东西——三组门全点亮后,这个容器本身就是。

01

架构:一循环两账

L3 电路 + 两本账
同一个八步循环(L3 电路),并行跑两本账。零售商账是欣和账的派生视图——把同一批欣和晶体管 + 一线收来的竞品数据,重新聚合成零售商语言。所以没有欣和底座,零售商账算不出来 → 决定了点门顺序(见 §4)。
账 A · 零售商品类记分卡
目标函数 = 门店总利润 + 份额。决策变量 = 各品类的角色/策略/战术。约束 = 货架空间 + 顾客一次到店。算法 = Fair Share → Point Gap → 机会(剔除衰退)。
账 X · 欣和 SKU 记分卡
目标函数 = SKU 生命力 + 动销 + 渗透(北极星 200M 瓶)。决策变量 = SKU 角色/生命周期阶段/投入。算法 = 心跳 r → node_target 缺口(自上而下)。
对应门 · 把两本账缝在一起(keystone)
每一步:先算零售商要什么 → 再算欣和能填什么 → 交集处出唯一一个动作。零售商收"缺口↓回到 Fair Share",欣和收"SKU 生命力↑动销↑"。这是"借力发展"的结构机制。

05

门 #1 实样:角色 → 4 战术

证明可搬,非概念
把最便宜的第 1 道门填满,作为"这套 spec 能落地、不是画概念"的证据。下面 4 张表直接搬自 ECR 个案原图(范围/价格/货架/促销 × 5 角色),工程化即 IF 角色=X THEN 规则=Y。
▸ 范围 RANGE
角色范围战术
目的性Complete Range:消费者期待的全有 + 额外品项;全部子品类/区隔/品牌/SKU
优先性Broad Range:至少和竞品一样宽 + 部分额外;全部子品类/区隔/品牌
基本性Basic Range:有竞争力的宽度,多数品项;全部子品类 + 主力品牌 + 主力 SKU
季节性Timely Range:全部子品类和区隔
便利性Select Range:仅主力品牌和 SKU
▸ 价格 PRICE
角色价格战术
目的性Aggressive:价值领导,全品类每包最优价
优先性Strongly Competitive:主要成分对竞品最优,次要成分与竞品持平
基本性Competitive:主要成分与竞品持平,次要成分溢价
季节性Competitive seasonally:部分成分贴近竞品
便利性Acceptable:每包在竞品 15% 以内
▸ 货架 SHELF
角色货架战术
目的性Best 位置;高客流、高停留、高空间分配
优先性Quality 位置;中客流、中-高空间
基本性Average 位置;平均客流、平均-最低空间
季节性Good 位置;高客流、平均空间
便利性Available 位置;低空间分配
▸ 促销 PROMOTION
角色促销战术
目的性Aggressive:高强度、高频次、多样类型、品类定制
优先性Build:高强度、高频次、多样类型、主力品牌/SKU 定制
基本性Maintain:中等强度/频次、常规类型
季节性季节性促销活动、品类定制
便利性Low 强度

门 #6 实样:7 策略 × 角色矩阵

角色筛策略菜单
用法两层:角色是品类级(矩阵筛出能用的策略菜单)→ 策略是区隔级(每个区隔按自己的份额/毛利/客单特征,从菜单里各挑一个,凑成均衡计划)。X = 该角色可用该策略。
7 策略目的性优先性基本性季节性便利性
Traffic 增来客XX–X–
Transaction 增客单XXX–X
Cash 现金流–X–––
Profit 提利润–XXXX
Turf 防御X––––
Excitement 刺激X––X–
Image 形象––––X
自洽性验证:个案品类=优先性,矩阵允许的正好是 {Traffic/Transaction/Cash/Profit};个案 8 个区隔分派的策略全落在这 4 个里,从没用到 Turf/Excitement/Image——矩阵与个案完全自洽,确认这就是真实用法。
