⚛️ 决策日志 MySQL Schema (草案 v1)
晶体管的另一半 · 与 SKU 6 卡通过 sku_id 反应
每条决策 = 一个晶体管开关 · 30/90 天自动回填
CREATE TABLE decision_log ( decision_id VARCHAR(32) PRIMARY KEY, -- DEC-{ts}-{role}-{seq} -- WHO 谁决策 decided_by VARCHAR(32) NOT NULL, -- 个体 ID (黄建国) role VARCHAR(16) NOT NULL, -- DOM/CTL/SR/ICA... org_scope VARCHAR(64), -- 沪西华南 -- WHAT 做什么 decision_type ENUM( 'invest','restructure','staffing', 'pricing','channel','sku_lifecycle'), decision_text TEXT NOT NULL, -- WHY 为什么 reason_text TEXT, trigger_event VARCHAR(128), -- 与 SKU 6 卡的关联 (核反应) affected_skus JSON, -- [{sku_id, impact_type}] -- IMPACT 影响 (预期 + 实际) impact_predict JSON, -- {r_delta, a_delta, roi} impact_actual JSON, -- 30/90 天自动回填 -- AUTHORITY 权限 authority VARCHAR(32), -- self/manager/board approved_by VARCHAR(32), -- 投入 budget DECIMAL(12,2), budget_source VARCHAR(32), -- 时间戳 + 学习状态 created_at TIMESTAMP NOT NULL, verify_30d_at TIMESTAMP, verify_90d_at TIMESTAMP, learn_status ENUM( 'pending','verified','failed','partial'), learn_score DECIMAL(4,2), -- 实际/预期 比率 INDEX (decided_by, created_at), INDEX (role, decision_type), INDEX (learn_status) );
⚛️ 化学反应公式
每条 decision_log.affected_skus 引用 sku_6cards.sku_id
每个 SKU 6 卡的"实际产出"自动汇总相关决策
30/90 天后自动回填 impact_actual + learn_score
→ 决策可学习 · SKU 可追溯 · AI 可进化
