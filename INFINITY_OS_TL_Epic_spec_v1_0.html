1数据结构 · 核心实体
目标：一套多态 schema 承载 ICA/SR/DOM 三级。对象层级用枚举区分，避免三套表。
  • biz_object：经营对象(门店/客户/区域)，多态 + 层级链 parent_id
  • sku：主数据 + 战略权重 + 生命周期阶段 + 阶段红线
  • transistor：SKU×对象×周期 的状态(核心表，逻辑门读它)
  • fee_structure：费用结构(来自大卖场_MT调研 + 财务台账)
  • jbp_contract：满分尺子(自上而下目标 + 责任人 + 审批模型)
  • causal_candidate：因果候选因 L2 表(含探针 SQL)
  • inf_learning_log：因果样本库(攒到~500训真模型)
层级即放大：transistor 不分 ICA/SR/DOM——SR 是对 CUSTOMER 级 transistor 的聚合，DOM 是对 REGION 级的聚合。聚合靠 parent_id 上卷，不是新表。
2晶体管 DB
目标：状态存储 + 三信号计算。r_actual 直取自汉询 rixiao。
-- 经营对象: 一套表跨 ICA/SR/DOM
CREATE TABLE biz_object (
  object_id  BIGINT PRIMARY KEY,
  level      ENUM('STORE','CUSTOMER','REGION') NOT NULL,
  name       VARCHAR(128),
  parent_id  BIGINT,            -- 门店→客户→区域
  owner_user BIGINT,            -- 责任人 ICA/SR/DOM
  cust_type  ENUM('KA','DISTRIBUTOR','HORECA') NULL
);

-- 晶体管: SKU×对象×周期 的状态(核心)
CREATE TABLE transistor (
  sku_id      BIGINT, object_id BIGINT, period DATE,
  fob         DECIMAL(10,2),     -- 出厂价
  cogs        DECIMAL(10,2),
  q_target    INT,               -- 自上而下·北极星分解
  q_actual    INT,               -- 汉询 rixiao 聚合
  m_contrib   DECIMAL(10,2),     -- 单瓶净贡献
  f_fixed     DECIMAL(12,2),     -- 固定通路费
  q_star      INT,               -- 规模门槛 F/m
  net_margin  DECIMAL(5,2),
  pull_share  DECIMAL(5,2),
  sig_margin    ENUM('G','A','R'),  -- 三信号
  sig_volume    ENUM('G','A','R'),
  sig_structure ENUM('G','A','R'),
  exec_done   BOOLEAN,           -- JBP动作执行?
  PRIMARY KEY(sku_id, object_id, period)
);

-- 费用结构(是/否来自调研; amount 待KAM/财务填)
CREATE TABLE fee_structure (
  object_id BIGINT, fee_code VARCHAR(32),
  charged   BOOLEAN,
  amount    DECIMAL(12,2) NULL,  -- 当前NULL·App倒逼填
  rate_pct  DECIMAL(5,2),
  cost_type ENUM('FIXED','VARIABLE'),
  invest_to ENUM('TRADE','CONSUMER'), -- 投大润发/消费者
  PRIMARY KEY(object_id, fee_code)
);

r_actual 来源：汉询 rixiao(日销)按周期 T 聚合 → a，r = a/T。Hologres 主查、MaxCompute 批量历史(供季节模型)。

-- Hologres T+1
SELECT sku_id, store_id, SUM(qty) a
FROM hanxun.rixiao
WHERE biz_date BETWEEN :t0 AND :t1
GROUP BY sku_id, store_id;
3逻辑门引擎
目标：IF-THEN 规则做成数据(规则表)不硬编码 + 2×2 + 优先级门。机制层不是概念层。
public interface GateEngine {
  // 单晶体管 → 裁决(规则来自 gate_rule 表)
  Verdict evaluate(Transistor t);
  // 2×2: 执行做没做 × 业绩好坏
  Quadrant diagnose(boolean execDone, boolean perfGood);
  // 优先级门: 缺口×战略权重×杠杆 → 排序
  List<Priority> rank(List<Transistor> portfolio);
}
-- 逻辑门规则即数据(可配置·非硬编码)
CREATE TABLE gate_rule (
  rule_id   INT, priority INT,
  cond_expr VARCHAR(255), -- "sig_volume='R'"
  verdict   VARCHAR(64),  -- 角色错位/砍买路钱/正样本
  action    VARCHAR(255)
);

2×2 → 原因类：没做×差=执行问题坐实；做了×差=投入无效·送因果引擎；没做×好=预算本身错；做了×好=正样本入 log。

优先级门评分：score = gap_qty × sku.strategic_weight × leverage。ICA 排"客户×SKU"，DOM 排"调人/调资源"——同函数，对象不同。

CREATE TABLE causal_candidate (
  cand_id  VARCHAR(8) PRIMARY KEY,  -- A1/C1...
  layer    ENUM('SUPPLY','VISIBLE','DEMAND','PEOPLE','EXTERNAL'),
  name     VARCHAR(64),
  probe    VARCHAR(255),

CREATE TABLE inf_learning_log (
  log_id BIGINT AUTO_INCREMENT PRIMARY KEY,
  logged_at DATETIME, sku_id BIGINT, object_id BIGINT,
  symptom VARCHAR(64), attribution VARCHAR(8),
  evidence TEXT, action VARCHAR(255), effect VARCHAR(255),
  confidence ENUM('HIGH','MID','LOW'),
  FULLTEXT(symptom, evidence, action) -- 替代 pgvector
);
护栏：/diagnose/confirm 只在归因确认时写 log——没确认不喂，防噪声污染未来因果模型。
