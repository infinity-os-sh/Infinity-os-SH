ICA V2.9 · L2 生命周期门 回填规格卡

署名 DS  ·  解决 满分/该救按开关裸数=阶段盲  ·  层 在 L1→满分 之间插一道 L2 门  ·  当前 S2 · 对外不可说 Agent
◆ 一句话目标 · 改什么 / 不改什么

目标：让满分/该救的标准随 SKU 生命周期阶段变，不再按「6 开关里几个 OFF」裸算。导入期动销 OFF=正常不扣分；成熟期动销 OFF=病危。

不动：L1 六开关本身、年级系数(奖励)、L7 收入/明细账/提现。本卡只在 L1 和满分之间加一道门，是定向回填，不是重写。

1业务契约WHO · READ · DECIDE · OUTPUT
WHO 触发
ICA 打开「店内」看板，或 INF-XD 盘点回传后——对该店每个 SKU 触发一次状态评估。
READ 读
该 SKU×店 的 ①6 开关 ON/OFF ②生命周期阶段 ③阶段满分标准表。
DECIDE 决
按该阶段「应 ON 开关集」算 满分/实得/差距，并定该救哪几个（只在阶段内应 ON 却 OFF 的）。
OUTPUT 出
阶段标签 + 阶段满分 + 实得 + 差距(落到具体开关) + 该救动作 + 病危红标(仅成熟/衰退动销 OFF)。收入额仍走 L7（年级系数另乘）。
2触发条件
  • 进店看板渲染时
  • INF-XD 盘点数据回传时（开关状态变）
  • 该 SKU 的生命周期阶段变更时（导入→成长 等）
  • 不触发：单纯切 Tab、看一本账——读缓存即可，不重算
3数据读取（输入字段）
标识
sku_id · store_id
开关
switches{上架,有货,陈列,动销,复购,毛利} 各 ON/OFF（L1，来自 INF-XD/手填）
阶段★
lifecycle_stage ∈ {导入,成长,成熟,衰退}——第一类字段。V2.9 过渡期可由现有「年级」临时映射(1→导入·2→成长·3→成熟·4→衰退/收割)，但应尽快独立成 SKU×店 粒度字段。
标准表
stage_standard_table（阶段→应 ON 开关集 + 权重）——配置层，见 Section 4。
奖励
年级系数（reward，已有，本卡不改，仅说明它和判断分开）
⚠ 阶段 ≠ 角色：角色(主推/试饮/维持·旗舰/防御…)是名词标签；生命周期阶段(导入/成长/成熟/衰退)是判断的尺子，两者别混(反模式②)。年级是消费者关系等级，和阶段相关但不等同，过渡期当代理可以，长期独立。
4决策逻辑 · 阶段满分门（核心）L2 GATE
生命周期阶段满分 = 这些全 ON（应 ON 开关集）该救 = 这里 OFF 的不算病危（是目标，非失分）
导入上架 · 有货 · 陈列（+试饮触达）这三里任一 OFF动销 · 复购 · 毛利
成长上架 · 有货 · 陈列 · 动销(上升)前四里任一 OFF复购 · 毛利（在建）
成熟全 6 开关 ON · 维持任一 OFF · 动销 OFF=病危—
衰退有货 · 复购 · 毛利（守存量）这三里任一 OFF动销下滑(预期)；评估退场
满分分母 = 该阶段「应ON开关集」的权重和
实得 = 应ON里·实际ON的权重和
差距/该救 = 应ON里·OFF的开关（只算阶段内的，不碰目标开关）
病危红标 = 仅当 成熟/衰退期 本应ON的关键开关(动销/复购)却OFF
收入 = 该救完成带来的状态提升 × 年级系数(reward·不变) × 店级×时段×CB（L7已有）
关键变化：满分分母随阶段变——禾然(导入期)的满分不再要求动销 ON，所以「动销没起来」不再被当成差距扣她的分。
5输出规格
  • 每个 SKU 卡：🏷️阶段标签(导入/成长/成熟/衰退) + 生命力分(按阶段满分算·非裸开关数) + 满分/实得/差距 + 该救清单(只列阶段内应ON却OFF) + 病危红标(仅成熟/衰退动销OFF)
  • 店级满分汇总 = 各 SKU 阶段满分之和（替换现在的 ¥664 裸算）
  • 差距文案要带阶段：「禾然(导入期)该救：有货 / 陈列 — 动销还没到，不扣分」
6自主边界
  • ICA 可自主：按该救清单去拨开关、调陈列、做试饮（执行权不变）
  • 阶段判定权不归 ICA：否则她会把难做的 SKU 标成「导入期」逃避考核。lifecycle_stage 由 BU/总部按 SKU×店 设定（或系统按上架时长+动销数据自动判·待 TL），ICA 只读
  • 标准表锁配置层：ICA 不能改阶段满分标准表（锁总部）
  • 报上游：阶段判定有疑义→DOM；应ON开关长期OFF且非她能解决(如断货)→DSR/沈总
↻实例对照 · 禾然 280ml（导入期 / 1 年级）
✕ 改前（V2.9 现状·阶段盲）

开关：上架● 有货○ 陈列○ 动销○ 复购● 毛利●

满分按全 6 算 → 3 个 OFF → 生命力弱 48

该救：有货 + 陈列 + 动销（3 个）

问题：导入期「动销 OFF」本是正常(刚铺·还在试饮)，却被当差距，误导美顾、压低她的分。

✓ 改后（接 L2 门）

阶段 = 导入 → 应 ON = {上架·有货·陈列}

实得：上架 ON（1/3）→ 满分按 3 项算

该救：有货 + 陈列（2 个）；动销○ 显示「目标·还没到·不扣分」

生命力按导入满分重算更公允；救活奖励仍 ×2.0(1 年级)不变。

对照成熟期 SKU（如六月鲜极鲜）：若它动销 OFF，照样红色病危——同一套门，不同阶段不同尺子。这就是修掉的阶段盲。
