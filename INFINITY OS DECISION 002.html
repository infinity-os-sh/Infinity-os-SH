DECISION-002 · 对齐/自主治理哲学 + 处置分级

AI 时代企业管理中 Alignment 与 Autonomy 的边界与执法纪律 —— 全员对齐基准
编号 DECISION-002版本 v1.0日期 2026-06-02署名 DS状态：待 DS 拍板锁定

01背景与触发

评估文件2「云端中枢控制台」时暴露出一个根本分歧：同样的实时数据，可以做成自动惩罚、自动划扣的控制者，也可以做成对齐结果、释放方法的守护者。技术一样，哲学相反。本决策把这条哲学边界、以及它落地为执法纪律的方式，钉成全员基准——所有「带执法的 Agent」据此而建，避免再凭控制者直觉做东西。

02决策正文（D-1 ~ D-7，具约束力）

D-1对齐与自主的根本判断
「既要对齐又要自主」的根本需求不变，变的是两者的换取关系。非 AI 时代二者此消彼长（靠监工换对齐、必牺牲自主）；AI 把对齐成本打到接近零，使二者可同时变大。机制是：对齐的锚点从「行为层」上移到「结果/状态层」（SKU 生命力）——不再管「你怎么做」，只对齐「SKU 该被拨到什么位置」。
D-2立场选择：守护者，非控制者
AI 拿掉了「因为看不见所以必须管」这个借口，于是「管松管紧」从技术约束变成价值观选择。INFINITY OS 选守护者。守护者不是更软，是更准、更狠地只打该打的人，与控制者的差别在三点：① 默认信任守规的多数（非人人嫌疑）② 正当程序（可申诉、看理由）③ 警察≠法官（AI 检测，人+双签裁决）。
D-3融合成立的两个开关（条件，非口号）
① 结果信号必须真实且可归因。否则 AI 规模化的不是对齐，是规模化的刷量（古德哈特定律）。② 人必须有值得做的判断空间。若 Agent 把每个动作都派死，自主归零，合伙人退化成接单零工。两个开关任一不成立，融合即不成立——「真实」与「守护者非控制者」是开关，不是道德口号。
D-4处置二维模型：程序=常量，力度=变量
处置不是「在松紧之间找一个点」，是一张二维函数表。两条轴独立、不互相交换：
程序 = 常量（一律拉满：检测≠判决 · 可申诉 · 双签裁决）
力度 = 变量 = f( 信号置信度 × 处置可逆性 × 是否惯犯/故意 )
力度可以比控制者更硬，同时程序一个不少——「硬，但讲程序」。绝不为了拿到「硬」而牺牲「程序」。
D-5判定门 + 力度阶梯（详见处置分级表 v0.1）
判定门（谁扣扳机）：高置信 + 可逆/低伤害 → 可自动；不可逆/高伤害 → 即便高置信也必须双签 + 申诉（误判不可逆=制造冤案）。
力度阶梯：L0 提示自纠 → L1 警告记录 → L2 扣分/暂缓 → L3 扣奖金/收权限(双签) → L4 退出/上报(双签+上级)，按惯犯度升级。
D-6威慑原理：确定性 > 严厉性
非 AI 时代被抓概率低（~5–10%），作弊期望收益为正，故钻漏洞是必然，非人品问题。AI 把被抓确定性拉满（→95%+），同样的罚，作弊期望收益即转负。故无需把罚加重到吓人（重罚只会误伤好人 + 引发反弹）——把「一定被抓」做实，威慑反而比控制者的自动重罚更强。
D-7漏洞优先判定为规则缺陷
若同一漏洞被多人触发，优先判定为系统/规则设计缺陷，回流重设规则，而非无差别罚人。失真检测的产物一半喂执法、一半喂规则重设。归因未明的异常（如动销骤降）只标记 + 转人工查因果，不自动处罚。

03适用范围与关系

强制适用于
所有「会处置 / 发激励 / 反作弊」的 Agent：反作弊、Uber 激励、财务（对赌违约/超预算）、HR（熔断/算薪）、派单互锁、二批商（窜货）等。
引用方式
每个执法 Agent 的 spec 卡 ⑥自主边界 引用本决策与处置分级表。
与宪法关系
不修改合伙人制宪法；本决策是宪法「守护者」原则的操作化落地，作为操作附录。
与 DECISION-001
扩展其双签机制——从「批花钱（审批流）」扩到「批处罚」；双签口径（总部 + 大区 GM）保持一致。

04前置依赖与已知缺口（诚实记录）
