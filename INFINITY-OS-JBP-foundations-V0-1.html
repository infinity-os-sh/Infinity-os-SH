◈ 每个 SKU = 一个合伙人节点

地基②③财务 Agent + HR Agent（JBP 的左右手）

JBP 下达卡③目标后，必须有人拨钱、有人派人执行——这就是财务和 HR。它们各自也要先定业务契约（像 JBP 这份），下面是「长什么样」的形状。
💰 财务 Agent
钱脉络 · 投资大脑 · owns 卡④
是什么
把钱按 SKU 生命周期阶段投对地方的大脑。
职责
按阶段分配投资（导入大→成熟收）· 守毛利红线 · 追踪「投了→有效无效→为什么」(投入有效性=因果)。
给 JBP
可用费用池 + 毛利红线。
边界
暂缓自动 · 放行/处罚双签（遵 DECISION-002）· 不自动改账。
🧠 HR Agent
人脉络 · 收入绑定大脑 · owns 卡⑤
是什么
把人的收入绑到 SKU 状态结果上的大脑（晶体管↔收入，收入是钩子）。
职责
据卡⑤产出算薪 · CB/AIQL 衡量执行质量 · 合伙人晋升/退出。
给 JBP
执行人 + 收入绑定（谁去做、做对了多赚）。
边界
算薪走分级+双签 · 不自动冻结权限（遵 DECISION-002）。
它们怎么咬合：同一个 SKU 6 卡节点上——JBP 写卡③（目标）、财务写卡④（投入）、HR 写卡⑤（产出），卡①②⑥是三者共享的锚。卡④和卡⑤一对比 = 「投了多少→换回多少→值不值」= 投入有效性 = 因果模型（最深缺口）。这就是 JBP—财务—HR 闭环合上的地方。
接下来的顺序（JBP 做真的前提）
1
SKU 6 卡 schema 先跑通——5–10 SKU + 六月鲜 1 BU + 6 个月（宁愿慢）。
2
财务 Agent + HR Agent 各定业务契约——像 JBP 这份 V0.1，先纸面审过再写。
3
JBP 接上三者 + 信息脉络——闭环合上，才从 Layer 1.3 演示变真 Agent。
