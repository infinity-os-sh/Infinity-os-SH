01 / FEEDBACK LOOP · 学习闭环
5步循环·Agent每天都在学
▸ Agent学习的5步闭环
1
判断·Agent给出建议
"建议补货1000瓶·置信度85分·原因A/B/C"
2
执行·人工确认+实际操作
销售经理批准 → 经销商实际进货1000瓶

结果·1周后真实数据回流
实际卖了多少·剩多少·有没有压货

评分·outcome_score自动计算(0-100)
预测准 = 高分 · 预测偏 = 低分
5
调整·Agent置信度自动升降
累积10次高分 → 同类决策置信度+5 · 反之-5
02 / SCORING · outcome_score计算公式
怎么判断Agent推单准不准
RULE · 推单评分
Omakase
推单后1周·自动评分
outcome_score = 100 - |实际销售 - Agent预测| / Agent预测 × 100 例:Agent预测补1000瓶 实际1周卖了: 900瓶 → score = 100 - |900-1000|/1000 × 100 = 90 (优秀) 700瓶 → score = 100 - |700-1000|/1000 × 100 = 70 (合格) 400瓶 → score = 100 - |400-1000|/1000 × 100 = 40 (差·学习) 1100瓶 → score = 100 - |1100-1000|/1000 × 100 = 90 (优秀)
RULE · 失真评分
蝴蝶效应
预警准不准·要看人工反馈
outcome_score = IF 销售经理确认"是真问题" → 100 IF 销售经理标记"误报" → 0 IF 销售经理"未处理" → 50 (中性) 例:蝴蝶效应Agent报A类压货预警 销售经理点"已确认·确实压货" → score = 100 销售经理点"误报·只是促销前备货" → score = 0 销售经理7天没看 → score = 50
RULE · 校准评分
A值校准
a值校准准不准·下周对账
outcome_score = 100 - |本周实际a - 上周校准的a| / 上周校准的a × 100 例:上周校准节点X的a值=50瓶/天 本周实际a值算出来: 52瓶/天 → score = 96 (准) 65瓶/天 → score = 70 (有偏) 100瓶/天 → score = 0 (大偏·学习)
03 / CONFIDENCE · 置信度自动调整规则
高分→升·低分→降
CONFIDENCE RULE 1
向上调整
连续10次outcome_score>=80 → 置信度+5
▸ 触发条件 同一类决策(如"上海KA门店补货推单") · 最近10次平均outcome>=80 · 自动升级置信度
▸ 效果 该类决策默认置信度+5(从80→85) · 当置信度>=90时·自动批准·不需要人工
CONFIDENCE RULE 2
向下调整
连续3次outcome_score<=40 → 置信度-10
▸ 触发条件 同一类决策连续3次拉胯 · 立刻拉低置信度·防止继续犯错
▸ 效果 该类决策默认置信度-10(从80→70) · 同时触发"复盘工单"·DS+Tech Lead分析根因
CONFIDENCE RULE 3
异常熔断
单次outcome_score=0 → 同类决策暂停24h
▸ 触发条件 推单完全失败(score=0)·或人工标记"重大误报" · 立刻熔断
▸ 效果 该类决策24小时不再推送 · DS+Tech Lead必须复盘后才能恢复
04 / SAMPLES · 学习样本累积阈值
DS提过的"500样本"门槛
SAMPLE THRESHOLD
规则解锁
不同样本量·解锁不同能力
样本数 < 50 → Agent全部决策必须人工确认 样本数 50-200 → 高置信度(>=90)的决策可自动·其余人工 样本数 200-500→ 中置信度(>=75)的决策可自动·低置信度推送 样本数 >= 500 → Agent进入"自主区间"·只推关键决策给DS 8周项目预估样本累积: Week 1-2:0样本(尚未上线) Week 3-4:~50样本(上线初期) Week 5-6:~200样本(全量推单后) Week 7-8:~500样本 → 进入自主区间
