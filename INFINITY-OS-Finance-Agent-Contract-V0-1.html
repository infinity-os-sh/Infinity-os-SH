钱脉络的投资大脑 · owns SKU 卡④阶段投入 · 先纸面契约，审过再写

财务 Agent 不是记账/核销工具，是把钱投对地方的投资大脑。钱脉络有两半：投入（卡④投多少、值不值）和现金流（回款/账期/资金占用——钱与货反向、有时滞）。它 owns SKU 卡④阶段投入，是 JBP—财务—HR 闭环里的「钱」那条脉络——给 JBP 费用池和红线，与 HR 的卡⑤产出一对比，就是「值不值」。

契约一WHO · 谁触发

节律触发
预算周期自动跑：年度/季度预算、每 round（7天）/月复盘投入（预预实节律）。
联动触发
JBP 发起某渠道计划时被调用——算「费用池够不够、红线守不守」。
偏差触发
预算消耗触及阈值（如 90%）或某 SKU 投入产出比异常 → 提示。
人工触发
财务负责人 / GM 发起预算分配或复盘。

契约二INPUT · 读什么（标来源）

SKU 阶段
SKU 角色 + 生命周期阶段（卡①②）——决定投资比例规律 SKU 6卡 schema
SKU 目标
该阶段要投到什么程度（卡③） JBP / schema
毛利红线
22%（任何投入不得突破） 财务基准
实际核销
实际花了多少 SAP · Dcloud · 卡 Step2
实际产出
动销 r 实际（算投入有效性用） 汉询 · 信息脉络
回款/资金
回款进度 · 账期 · 资金占用（压货占资） SAP · 一本账
既有模型
年级费用率 / 全链费用率公式 / 三维预算体系 现有财务模型

契约三DECIDE · 怎么算（核心）

投入分配
按 SKU 生命周期阶段定投资比例——有规律：导入期投入比例最大 → 成长 → 成熟收敛 → 衰退/退场最小。
守红线
毛利红线按生命周期加权，不是平铺：成熟/跑量 SKU 严守 22%；战略/导入期 SKU 允许阶段性低于红线（投资期），但受亏损上限约束（额度待 DS 定）。
预预实
预算 · 预测 · 实际三比 → 算偏差。
投入有效性
卡④投入 vs 卡⑤产出 → 算 ROI / 有效无效 → 喂因果模型。
现金流
监测回款健康 / 账期成本 / 资金占用（压货占资），预警资金风险。
中台分摊
中台费用按规则分摊到各 SubCo/品牌/渠道，roll-up 出 SubCo P&L（口径待 DS 定）。
安全阀
预算消耗触阈值 → 建议收紧/下调悬赏（建议，不自动改账）。
投资比例规律（示意）：导入期＝高投入抢渗透铺货 · 成长期＝中高投入扩量 · 成熟期＝收敛保利润 · 衰退期＝最小维持或转防御。具体比例待真实数据校准，不编假精度。
红线（投入有效性的诚实边界）：现在只能算相关性（投入高+产出高），不能归因「为什么」（是投入有效，还是季节/竞品/真实需求？）。归因要等因果模型——这是最深缺口，财务 Agent 也卡在这。

契约四OUTPUT · 推什么

主产出
写入 SKU 卡④阶段投入（投多少、投哪）。
给 JBP
该渠道/SKU 可用费用池 + 毛利红线（JBP 算对赌时用）。
复盘
预实差报告 + 投入有效性评估（哪些投入有效/无效）。
预警
预算超阈值 → 建议收紧（待审批，不自动执行）。
推给谁
财务负责人 / GM / 相关 BU；改账/放行/收紧均为建议，需双签或走审批流。

契约五自主边界（引用 DECISION-002）

能做
算投入分配建议、预实对比、有效性评估、预警提示、写卡④。
不能做
不自动改账、不自动放行超预算、不自动扣款。
审批
投入/折扣/超额审批走 DECISION-001 五模型 + 三维阈值矩阵——财务是审批流的主场。
处置
暂缓可自动（可逆）；放行/超预算/处罚走 DECISION-002：双签 + 转人工 + 可申诉。

接线JBP—财务—HR 闭环

财务（钱 · 卡④）给 JBP（货+促销 · 卡③）费用池和红线 → JBP 下达目标 → HR（人 · 卡⑤）派人执行 → 结果回流 → 财务用卡④ vs 卡⑤算投入有效性 → 校准下一轮预算。卡④和卡⑤的对比，就是因果模型的入口。

地基依赖与缺口（诚实）

必须挂 SKU 6 卡 schema — 财务写的是卡④，schema 未建则无处可写。
因果模型待建 — 「投入为什么有效/无效」现在答不了，只能算相关。这是财务 Agent 的天花板。
核销数据卡 SAP Step2 — 待业务填 r_target + grade 导 SQL 解锁。
现状 — 六月鲜 SubCo CFO 财务智脑有对话原型（已建），但「按生命周期分配 + 有效性学习」未到位 = Layer 1.3。
