## 算法落点(SKILL §2)
- **四维三权**:买得到/看得到/动销 进加权(§2.3 权重表·仅按阶段);**毛利是闸不是维**(§2.4 破线封顶 CAP=60)。
- `score = Σ(w_i·r_i)×100/Σw_i`(§2.2 归一化,同时吃掉 §2.5 no_data 维的权重重分摊),毛利破→`min(score,60)`。
- `gap_i = w_i·(1−r_i)`(原始权重·只在有数据维上算)→ `top_gap` → 战术映射(买→补货/范围L5-07·看→货架大卖场②③+INF-VL·动→促销+价格)。
- 任务种子三类(§5 inbox):**无差 / 待批 / 已派**;触闸/疑压货/低置信 → 待批(人工核)。**只建议非指令**。

## 自主边界守住(SKILL §1⑥ / §5)
- **只算只建议**:不改目标/评分表、不派钱、不判退场、不改权重表(人来调)。`task_seed.note` 明示「建议·非指令」。
- **判级类**:`no_data ≠ 0`,缺维不计 0、权重分摊重归一、降置信度,绝不拿缺数据当差。
- **tracker 校验**:毛利闸 / 压货(铺货远超目标→疑压货,标待人工核+降置信)/ no_data 置信 / 权重和。
- **观测留痕**:`flags` 标用了哪套政策(`SKU_SCORE_pending`)、缺哪些维、是否疑压货,可审计回溯。
