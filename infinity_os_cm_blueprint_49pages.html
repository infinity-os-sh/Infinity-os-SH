品类管理 逐页剥开

把这份 ECR 品类管理个案(联合利华×零售商,49 页)层层剥开,每页的文字和表格都转成"我们的 Agent 怎么用"——门、数据、动作。这是品类管理 Agent 的主干蓝图。

ECR 原文(蓝)= 它怎么说我们怎么用(橙)= 转成 Agent 的门/数据/动作
开场(第1-4页):个案目标=理解 CM 模式、各步骤的关联、用一份品类计划串起实操(P1-2)。背景=联合利华与某零售商联合先导,经济部/ECR Taiwan/PWC 支持,验证国际标准能否用在台湾,成果出版(P3)。数据=真实 SKU 与市场,为保密做了脱敏(P4)。
→ 我们用:这正是我们做的事的"祖本"——把这套人工+周期性的咨询流程,工程化成连续、少人干预、自动关差的 Agent。
模型国际品类管理模式(八步循环)P5
ECR(P5): 八步轮:品类定义 → 角色 → 评估 → 评分表 → 策略 → 战术(商品组合/价格/促销/货架)→ 计划执行 → 品类分析(检视),循环。
我们用: 这就是 Agent 的 L3 电路——八步串成闭环,每步一道门,检视回到评估,持续转。我们的"一循环两账一组门"就架在这上面。
①品类定义P6-8, 31
页ECR 原文要点我们怎么用
P6定义与角色由供应商和零售商管理层事先达成共识(讲义3-4页)账X×账A 在第①步就要对齐品类树;欣和 SKU 挂到零售商品类节点上,双方共识是双赢的前提
P7品类架构树:衣物清洁→洗衣用品→主清洗剂(传统/浓缩/浓缩洗衣精/冷洗精)+柔软精+漂白+预洗+熨烫对应我们缺的 segment_code 品类树主数据 + "SKU→系列/子品牌→角色"映射表(六月鲜那一刀证明:没有这张表,系统认不出遵循自然/轻)
P8今日聚焦:浓缩类洗剂(在树里圈出一个区隔)Agent 支持下钻到任一区隔单独跑(我们做六月鲜就是这个动作)
P31定义全文:消费者洗衣过程用的产品与服务,把脏衣变干净清爽平整每个品类树节点带一句消费者价值定义(为后面"乐得买"因果链用)
②品类角色P9-11, 32
页ECR 原文要点我们怎么用
P95 个角色:特色/目的性、优先、基本/例行性、临时/季节性、附加/便利性角色门的标签集。注意层级:这是零售商给品类贴的;欣和给自己 SKU 贴的是六卡(7个),不混
P10本品类=优先,因为对业绩+毛利的贡献 加上 竞争重要性角色推导的三因子:业绩×毛利×竞争性(我们现有数据只有业绩,毛利/竞品待补=stage-2)
P11分布:特色10% / 优先25% / 基本50% / 季节5% / 便利10%;特色极少,因维持成本高% = 预算/约束,不是分类器。已验证:硬套到 SKU 清单出垃圾(77%死货)。它的真义="高投入角色只能少数"——和你六卡阶段投入律同源
P32优先角色全文:零售商要强化忠诚、增客流客单、愿牺牲毛利迎战;消费者高频、要最优价值与稳定可得角色自带双视角定义(零售商+消费者)→ 决定该角色下 4 战术的力度(P25-29)
六月鲜实证:角色不在数据里,必须算/定。我们用"自上而下战略意图 + % 约束"定了六月鲜内部:有机松茸=特色、遵循自然=优先、老鲜味=基本(现金牛)、老生抽老红烧=衰退退场。
③品类评估P12-17
页ECR 原文要点我们怎么用
P12评估三问:品类与各区隔现状?vs 市场如何?有何改善机会?评估门的三个子问=自身/相对竞品/机会,逐一对应一道计算
P13诊断:占有率降低、低于零售商整体销售、但利润仍维持三态信号:份额↓ × 整体对比↓ × 利润持平——这正是"输份额守利润"的典型,Agent 自动打这个标
P14区隔评估(销售):标出销售较差的区隔晶体管:区隔×业绩 达/不达
P15区隔评估(利润):标出明显改善机会晶体管:区隔×毛利 达/不达(我们这份大润发数据缺毛利,待补)
P16气泡图:零售商市占率 vs 市场成长率,四象限 Winners(HH)/Sleepers(HL)/Opportunity Gaps(LH)/Questionable(LL),泡大小=区隔规模;只有一个 Winner 且小这是评估门的核心可视化=份额×成长 2×2。直接套你六月鲜:有机/遵循自然=高成长(右),老线=低成长(左)→ 自动定象限
P17四个最大区隔:都有明显提升销售的空间(落在 Opportunity Gaps)机会门:大且低份额高成长的区隔=优先关差对象
缺口:这一步全要账A(零售商整品类+竞品)——大润发那份只有欣和自家出货,所以气泡图的"市占率"现在算不出,要靠一线收竞品(INF-XD)+第三方大盘。
④评分表P18-20, 33
页ECR 原文要点我们怎么用
P18品类整体"顶线"目标,必须与高阶主管讨论并实际允诺这就是你 6-04 的红线:目标=自上而下承诺,不锚实际。Agent 的 node_target 门
P19目标:448.6m、5.3%、35.0m、7.8%;已 +16.5% 成长,但零售商还想要更多目标从战略往下压,不为达成而调低(正是大润发那份目标设到去年67%的反面教材)
P20区隔评分表:120.0m/5.3%/4.9m/4.1%。"该不该在超过 Fair Share 的区隔抢更多份额?当然要,但不牺牲毛利"Fair Share 门(应得份额)+ Plan vs Target 两条数;约束:抢份额不丢毛利
P33评分表全文:夺回份额+维持现金获利;策略=四种选择性使用评分表 = 角色下的量化满分尺子,后面每个战术动作都回这张表校(P45)
⑤品类策略P21-24
页ECR 原文要点我们怎么用
P22给每个区隔分派不同策略,以求均衡的经营计划策略是区隔级分派,不是整品类一个;凑成均衡组合
P23每个品类角色用不同的策略组合(策略×角色矩阵)策略门=角色筛菜单。优先角色→{增来客/增客单/现金流/提利润}(已验证与个案自洽)
P24高占有率区隔→提利润;高单价+关联销售区隔→增客单区隔特征→策略的 IF-THEN 规则:份额高→Profit;单价高→Transaction
⑥品类战术(解决的主体)P25-44
这一段最长、也最关键——这里才是"解决",不是诊断。四个战术构面:范围/价格/促销/货架,各按角色出规则 + 个案真做了哪些动作。
页ECR 原文要点我们怎么用
P25综合角色+评分表+策略,确认四构面战术:范围/价格/促销/货架战术门入口:四构面=我们的 4P
P26范围战术(优先)=备宽广 SKU(Broad Range)角色→范围规则(表已搬入门#1)
P27价格战术(优先)=非常有竞争力(Strongly Competitive)角色→价格规则
P28货架战术(优先)=优良位置+略高于平均的空间角色→货架规则
P29促销战术(优先)=用主力品牌/SKU 做量(Build)角色→促销规则
P30策略摘要(讲义15页)=管理品类的蓝图,后续所有行动决策的参考基准这张蓝图=Agent 出每个动作时的校验基准
P34优先角色的完整战术陈述(范围≥竞品/主成分最优价/优良位置中流量高空间/高频多样促销含主力定制)门#1 的完整成文规则(已搬)
P35范围检视:竞1有19个SKU,本店14个,够吗?(讲义16页)范围门:本店SKU vs 市场/竞品宽度
P36SKU 按销售份额排名,零售商 vs 市场;前15个SKU=95%市场;3个市场热销没进店;本店最差2个市场也差范围门算法:头部少数扛绝大部分(同你大润发 10% 扛94%)→ 补市场热销缺的、砍两头都差的
P37范围建议:增 Wulin/LanPao/ColorCare(+3.09/+1.90/+1.78m),删 Wulin refill/Tide(−0.47/−0.11m),净 +6.19m / +5.3% / +5.46%份额范围门输出=带金额的增删清单(这就是六月鲜死货该删/该补的真实动作模板)
P38促销检视(频率):列每个零售商与竞品的促销频率/型式/价格(讲义18页),注意频率与组合差异促销门输入:频率×型式×价位 vs 竞品
P39促销频率:没有完美答案;组合需调整才有竞争力;把品牌促销频率对齐市占率是好起点促销门规则:促销资源份额对齐市占率(我们 demo 里 keystone 用的就是这条)
P40促销检视(价格):几乎所有促销价都高于竞品(讲义19页)促销价门:促销价位 vs 竞品差
P41促销影响:价格弹性与促销组合"并不容易被了解",建议"估计冲击、衡量下次实际结果"这是因果墙的原文承认。我们的因果门(门#7)同样:先估→执行→量实际→喂回学习,养而非速成
P42价格检视:相比竞1还行,但货架上主要区隔商品价格竞争力不够(讲义20页)价格门:主成分价 vs 竞品(分主/次成分)
P43所有价格现在皆具竞争性,同时无毛利损失(蓝色=调整后)价格门输出=调价清单(更有竞争力且不丢毛利的解,带前后对照)
P44货架检视:Spaceman/Apollo 类软件;平衡"供货天数";用最小排面+最高DOS推删减;余空间给支撑区隔策略的SKU货架门:DOS×最小排面规则→删减+补位建议(对接 INF-XD 现场盘点)
这一段就是"系统替你关问题"的本体:范围门出增删清单、价格门出调价清单、促销门出对齐市占的频率清单、货架门出排面调整——每条都是带金额、带前后对照的动作,不是报告。
⑦计划执行P46
ECR(P46): 向每个执行人充分沟通宗旨/目标/策略;双方共同定执行时间;店内实测货架调整并修正;持续检查 check-check-check,确保同意的动作真做到。
我们用: 这正是 派单互锁25步 + SFA + 抢单/CB——把战术门的动作派给具体的人(美顾/SR/DSR),系统记录、盯执行,人为裁量降到最低。
⑧检视(回到起点·持续改进)P45, 47
页ECR 原文要点我们怎么用
P45调整计划:每个战术决策都改变了销售与利润;累加变化值,和评分表原始目标比,据此调整,与管理层达成共识检视门:Σ各动作的预期影响 vs 评分表目标 → 缺口关没?差多少回去补
P47执行后检视:实际 vs 计划;分析理解偏差;重启整个流程(重做评估、确认或修订目标)。记住:品类管理是一个过程,不是一次结案的项目这就是关差闭环 + game-round 节律 + 喂因果学习(E1-E5)。"过程不是项目"=Agent 持续转,不是一锤子报告
这页是全篇的灵魂,也是你叫停我的那一刀:只诊断不闭环=把过去"人拍脑袋+不透明"的病换个壳。八步必须转起来、自动关差,才是 Agent。
结尾(P48-49):个案结果(最终评分表)+ 致谢。
→ 我们用:个案结果那张表 = 我们"双赢品类建议书"的成品长相参照(目标 vs 计划 vs 实际,带每条战术的贡献)。
